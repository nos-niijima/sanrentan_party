# サンレンタン — デザイン定義ダイジェスト

claude.ai/design プロジェクト `20557f91-cb19-4e4f-92ec-51d3d7ec62a3` から取り込んだ画面定義の精読結果。
各 `.dc.html` は `x-dc` カスタム要素 + `sc-for`/`{{ }}` テンプレート（DCLogic.renderVals() がデータ供給）で
モバイル枠（390×838px、上部にステータスバー 9:41）や共有ディスプレイ（1280×800）を描く宣言的モック。
**準拠テストの核は「画面に出る日本語コピー/ラベル」**。下記はその抽出。

> **招待リンク仕様 (2026-06-24, URL 分離)**: 招待リンクの正規パスを `/join/<id>` に分離した。
> - **`/rooms/<id>`**: ホストまたは着席プレイヤー専用の room 操作画面。未着席ユーザーが直接来た場合は `/join/<id>` に `router.replace` される（互換 fallback）。
> - **`/join/<id>`**: 招待リンクの landing page。未着席ユーザー向けの入室フォーム (JoinScreen) 専用。既着席なら `/rooms/<id>` に `router.replace`、`?j=1&c=<hex>` 付きなら auto-join → 成功後 `/rooms/<id>` に push。
> - **InviteCard コピー対象**: `/join/<id>` を指す絶対 URL（`window.location.origin + "/join/" + roomId`）。
> - **ゲスト signIn callbackUrl**: 現在 URL が `/rooms/<id>` でも `/join/<id>?j=1&c=...` に正規化される。auto-join は `/join` 側の useEffect で一元処理。
> - **`/browse` 入室フォーム**: 入力 URL から `(?:join|rooms)\/([^/?#]+)` を抽出 → `/join/<id>` に push（旧 /rooms/<id> 形式の URL も受け入れる）。
> - **狙い**: 「招待リンクをコピー」UI が嘘でなくなる。受け手の着席状態に応じて自然に分岐する。
>
> **実装上の判断 (2026-06-24, ユーザー判断)**: 『あいことば（部屋コード `#XXXX`）』のコンセプトは撤去し、
> 招待動線は **「招待 URL コピー」1ボタンに収束**。理由 — 機能しないコード入力 UI を頑なに残すより、
> 機能する UI を作る方針。影響範囲:
> - ホスト管理 ①「部屋を立てる」: あいことば項目・コード横コピーを撤去（CTA「この卓をひらく」のみ）。
> - 各 compact invite bar (②③④⑤): 「あいことば」「#XXXX」「コード横コピー」表示を撤去。
>   招待 UI は「招待 URL コピー」1ボタン + 「招待リンクを表示」展開トグルだけに簡素化。
> - /rooms/new: 「あいことば（placeholder #●●●●）」項目を撤去。
> - /browse: 「招待リンク / 部屋コードで参加」入力欄を「**招待リンクで参加**」のみに簡素化。
>   コード4文字入力は撤去。
> - 入室画面 (A/B/C 各案): 部屋情報内の「部屋 #XXXX」表示も整合のため撤去。
> 下記の各節では更新後の状態を反映している。

## 2. サンレンタン_画面1_入室.dc.html （入室 / プレイヤー）

- **目的/役割**: プレイヤーが部屋に入る入口（player、lobby 相当だが「途中参加OK」=進行中でも入れる）。**3レイアウト案 A/B/C**。
- **共通コピー**: 「Board Game Trial にもどる」、卓「たけの試遊卓」、「ホスト たけ」、ステータスタグ「第3R 進行中」、「いま4人が参加中」、「途中参加OK」。（『部屋 #XXXX』表示は 2026-06-24 にあいことばコンセプト撤去とあわせて削除。）
- **レイアウト A（ウェルカム式）**: ヒーロー「サンレンタン」「価値観をあてる三連単パーティ」→ 部屋カード → 入力ラベル「チーム名 または 名前」（値「チームたぬき」7/12）→「あなたの駒色」カラー選択（✓付き先頭）→ CTA「**入場してこの卓につく**」+ 注記「途中参加・途中回答OK。予想はホストが正解を出すまで提出できます。」
- **レイアウト B（入場券式）**: チケット「ADMISSION / 入場券」、「お名前（チーム名）をご記入ください」、駒色「駒色をえらぶ」、ミシン目下に CTA「**もぎって入場する**」、注記「予想はホストが正解を出すまで何度でも提出・修正できます」。
  - **rejoin 追加セクション (2026-06-24)**: ブラウザクラッシュ等の再入場救済として、入場券の下に「**すでに参加している場合は戻る**」セクションを追加。`view.players.filter(seat!==hostSeat && name)` が空でなければ表示し、各既参加プレイヤーの駒色ドット + 名前 + 「**これは私です**」ボタンを並べる。click 動作: 未ログイン→ `signIn('guest',{name:existingName, callbackUrl:/join/<id>?j=1&c=<existingColor>})` でゲストサインイン経由 → /join 側 auto-join が同 userId で backend.join() を呼び、`room.service.join()` が既存 RoomPlayer を見つけて same seat / same color を返す（rejoinSameSeat）。ログイン済（別人）は confirm のうえ性善説 fallback。注記「※他人の名前を選ばないでください」を併記（なりすまし防止は backend 未実装の既知制約、ゲムマ会場前提）。host (進行専任) はリストから除外。
- **レイアウト C（パドック名簿式）**: アプリバー「サンレンタン」、見出し「出走名簿」、「参加中のプレイヤー」リスト（きつね組[回答済み]/ふくろう亭[予想中]/はりねずみ[回答済み]/こぐま（ホスト）[ホスト]）、自分の行「あなたの名前」[記入中]、「駒色をえらぶ」、CTA「**名簿に加わって入室**」、注記「いつでも入室OK・進行中のレースから予想に参加できます」。
- **操作要素**: 名前テキスト入力、駒色選択（4色）、入場ボタン。
- **状態バリエーション**: A/B/C は意匠案。実装は1つ選定する想定。

## 3. サンレンタン_画面2_ホスト管理.dc.html （ホスト管理 / ホスト）

- **目的/役割**: 部屋と進行を司るホストの操作卓（host）。アプリバーはウォルナット木目＋赤バッジ「ホスト操作卓」。**5状態フレーム**。
- **① 部屋作成（初期）**: 見出し「新しい卓をひらく」「あなたがこのレースのホストです」、ラベル「卓の名前」(空入力欄。placeholder/初期値なし)、トグル「リンクを知っている人が参加 / 途中参加もいつでもOK」、CTA「**この卓をひらく**」のみ。（旧『あいことば（部屋コード）#4F2A + コピー』ブロックと旧副 CTA『プリセットを用意して始める』は 2026-06-24 撤去 — 招待は URL コピー1本に収束し、プリセット副 CTA は遷移先未確定の dead button だったため product から削除。プリセットへは②のタブ切替で到達する。さらに 2026-06-24 ユーザー指示により卓の名前 input の placeholder/初期値「たけの試遊卓」も撤去。）
- **共通: 各フレーム invite UI**: 「**招待 URL コピー**」ボタン1つ + 「招待リンクを表示」展開トグル（URL 表示用）のみ。あいことば／コード横コピーは出さない。
- **② 出題作成（新規）**: タブ「新規作成 / プリセット」、ラベル「お題」(値「BBQで肉以外に焼きたいのは？」)、「選択肢（出走）」「6つ」「3着まで選ぶので最低3つ」、ドラッグ可能な選択肢行（番号バッジ+名前+×削除）、「＋ 選択肢を追加」、CTA「**選択肢を公開して予想スタート**」+ 注記「公開後にあなたの1〜3着（正解）を選びます」。
- **③ プリセット**: 検索「登録した出題をさがす」、プリセットカード（タイトル+チップ+「＋{n}」+「使う」ボタン）、「＋ プリセットを新規登録」、注記「「使う」を押すと出題エディタに読み込まれ、公開できます」。プリセット例: 「BBQで肉以外に焼きたいのは？」「無人島にひとつ持っていくなら？」「最強の朝ごはんは？」「旅行で泊まるなら？」。
- **④ 正解選択（=締切トリガー）**: お題バナー + 注記「あなたの1〜3着を選ぶと予想が締め切られ、答え合わせされます」、選択肢行（バッジ「1着/2着/3着」or「えらぶ」）、プレビュー「あなたの本命（正解）」「③ → ① → ⑤」、提出状況「4人 提出済み ／ 1人 未提出 / 締切後は変更不可」、**赤** CTA「**🔒 正解を公開して締め切る**」。
- **⑤ 公開後の進行**: ステータス「第3R 結果公開済み」、完了バナー「第3R の払戻が完了 / 正解 ③→①→⑤ ・ 5人に配分」、「このレースの結果」ランキング、注記「次のレースは、あなたも選択肢を選んで予想に参加できます」（♘）、フッター「結果再表示」+「**次のお題を出す**」。
- **重要セマンティクス**: 正解選択＝予想締切＆採点トリガー。ホストは出題者で進行専任、次レースは参加もできる旨が明記。

## 4. サンレンタン_画面3_予想受付.dc.html （予想受付中 / プレイヤー · open）

- **目的/役割**: プレイヤーが 1着→2着→3着 を予想して「馬券を買う」（player、state=open）。**3レイアウト案 A/B/C**。冒頭に枠番カラー凡例「枠番カラー（選択肢＝出走馬）」。
- **共通コピー**: アプリバー「サンレンタン」、「第3R」、お題「BBQで肉以外に焼きたいのは？」、ステータス「予想受付中」、「⏳ ホスト確定待ち」、「出題 ホスト・たけ」。（『部屋 #XXXX』表示は 2026-06-24 撤去。）
- **レイアウト A（馬券マークシート式）**: 指示「出走から1着→2着→3着を予想」「タップで順に指定」、選択肢リスト（バッジ「1着/2着/3着」or「指定」）、下部に「三連単 馬券」スリップ（1着→2着→3着の枠チップ、「確定まで買い直し可」）、CTA「**この目で馬券を買う**」。
- **レイアウト B（ターフ＆表彰台式）**: お題ボード + **表彰台**（1着中央=「🌹 本命」, 2着左, 3着右。未選択は「?」）、ターフ「出走ゲート」「タップで台へ / もう一度で取消」、選択済みは「{n}着 に出走」表示。submit「現在の馬券 ③ → ① → ?」、未完成 CTA「**3着を選ぶと購入できます**」（グレー無効）。
- **レイアウト C（スリップ常時表示式）**: 上に「三連単 馬券」スリップ（3スロット、未選択は破線「未選択」、「あと1つで完成」）、「↓ 下のカードをタップして枠に入れる」、選択肢 2列カードグリッド（状態「{n}着 に選択中」or「枠に入れる」）、CTA「**馬券を買う**」。
- **操作要素**: 選択肢タップで 1→2→3 着割当（再タップで取消）、3着揃うと購入可。確定まで買い直し可。
- **状態バリエーション**: 馬券 0〜3 個選択。3個未満は購入ボタン無効（コピー「3着を選ぶと購入できます」「あと{n}つ」）。

## 5. サンレンタン_画面3_結果発表.dc.html （結果発表 / プレイヤー · revealed）

- **目的/役割**: ホスト確定後、正解と自分の馬券を照合し成立役・払戻・累積順位を見せる（player、state=revealed）。**2レイアウト案 1/2**。成立役は最高位1つだけ。
- **共通**: 「第3R」「結果確定」、お題、正解=①やきそば→②ほたて→③マシュマロ（例）。
- **レイアウト 1（払戻票式）**: フェルト上に**正解表彰台**（「🌹 1着」バッジ、各台に駒名）→ 払戻チケット（役ヘッダ「**サンレンタン！**」「1〜3着すべて順位までぴったり」「+6 点 払戻」+ 自分の馬券に「的中」スタンプ・各駒に✓）→ 役早見バー（サンレンタン6/サンレンプク4/ニレンタン3/プクプク2/タン1）→「部屋の累積順位」（3レース終了, 自分行ハイライト[あなた], ホスト行[ホスト]）→ フッター「次のレースを待っています / ホストが次のお題を公開します」+「待機中」。
- **レイアウト 2（電光掲示板式）**: お題ストリップ → LED ボード「◉ RACE RESULT」「確定」（1/2/3 着 + 「◎ 的中」）→ 自分の役ストリップ「あなたの成立役 / サンレンタン / +6 点」→ 配当表「配当（役）」「成立は最高位の1つだけ」（成立行は「成立」マーク）→「部屋の累積順位」→ フッター「**次のレースを待つ**」。
- **採点コピー（配当表の how）**: サンレンタン=順位までぴったり / サンレンプク=3つ当たり(順不同) / ニレンタン=1・2着がぴったり / プクプク=2つ当たり(順不同) / タン=1着がぴったり。
- **状態バリエーション**: 的中（役成立, +点, 「的中」スタンプ・「成立」マーク）/ ハズレ（役なし）。
